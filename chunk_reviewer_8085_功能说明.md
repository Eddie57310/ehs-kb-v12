# Chunk 数据质检工具 — 8085 功能说明

**程序**：`chunk_reviewer.py`  
**端口**：8085  
**项目路径**：`~/doc_parser_v11/`  
**整理日期**：2026-06-12

---

## 一、定位：入库后的「手术台」

8085 是 **已入库 chunk 的质检与修补工具**，与 8084 形成前后分工：

| | **8084 md_reviewer** | **8085 chunk_reviewer** |
|---|---|---|
| 操作对象 | `reviewed_md/*.md`（入库**前**） | `chroma_db` 已入库 **chunk**（入库**后**） |
| 主要场景 | 制度、规范等常规文档 | EHS案例、排雷库、设计部案例等 |
| 改动落点 | 改 MD 文件 | **直接改向量库**（删旧块 + 重嵌入） |

启动：

```bash
cd ~/doc_parser_v11 && venv/bin/python chunk_reviewer.py
# 浏览器：http://127.0.0.1:8085
```

**核心原则**：在 8085 保存 = 立刻影响飞书检索与问答，无需再跑 `index_reviewed_md.py`。

---

## 二、改动如何落库

点「保存并重新嵌入」或「确认完成」时：

1. 按 chunk id 删除旧记录
2. 用新正文重新计算 **bge-m3 embedding** 写回
3. metadata 更新 `modified_at`（确认时再加 `confirmed_at`）
4. 后台线程重建 **`bm25_index.pkl`**
5. 飞书服务约 10 秒内 **热加载** 新 BM25

## 三、界面布局：四栏工作台

```
┌──────────────┬──────────────┬──────────────┬──────────────┐
│ ① 源文件      │ ② MD 原文     │ ③ 编辑        │ ④ 飞书预览    │
│ 全页+框选截图  │ 只读对照      │ 改正文/导航   │ 模拟手机宽度  │
└──────────────┴──────────────┴──────────────┴──────────────┘
                    ↓ 底部：🗑 回收站（删块可恢复）
```

### ① 源文件栏

- 加载当前 chunk 对应 **整份源文件所有页**，可纵向滚动
- 图片来源：
  - 有 `image_path` → `slide_images/.../p0001.png` 等预渲染图
  - 无 `image_path` → 从 `Local_KB/` 源文件现场 PyMuPDF 渲染
- **鼠标拖拽框选**任意页 → 裁剪存 `user_crops/`
- 支持 30%–800% 缩放；多页独立 canvas
- 截图历史存 `user_crops/index.json`，key 为 `source::p页码` 或 `source::seqN`

### ② MD 原文栏

- 若 `reviewed_md/` 有对应 MD，显示只读全文
- 显示当前块章节路径（从 `【xxx > yyy】` 前缀提取）
- 用于入库后改 chunk 时对照原文

### ③ 编辑栏

- 可编辑 chunk 正文
- 显示：格式类型、页码、chunk id、确认/修改状态
- 上一块 / 下一块；撤回 / 重做

### ④ 飞书预览栏

- 实时模拟飞书显示效果
- 自动识别：**纯文字** / **Markdown 表格** / **HTML 表格**
- 对表格给出飞书兼容性警告（MD 表格不渲染、HTML 会乱码）
- 可切换 **iPhone 430px** / **Fold7 768px** 宽度

## 四、五大核心操作

### 1. 浏览与审核进度

顶部下拉列出 chroma 中所有 `source`，带进度标记：

```
[22块] 设计部/案例/xxx.pdf           ← 未确认，排最前
[15/48块✓] 设计部/案例/yyy.pdf       ← 部分确认
[✓ 70块] 设计部/案例/zzz.pdf         ← 全部确认，沉底
```

chunk 按 `page` 或 `chunk_seq` 排序。

### 2. 保存并重新嵌入

- 改正文后点「保存并重新嵌入」
- 写 ChromaDB + 重算向量 + 触发 BM25 重建
- 适用：改错别字、删噪音、调整排版、插入 `[📷 ...]` 图标签

### 3. 确认完成（批量质检流）

点「✅ 确认完成」= 保存 + 写入 `confirmed_at`：

- 自动跳下一个未确认块
- 当前文件全部确认 → 自动跳下一个未确认文件
- 全部完成 → 「🎉 全部文件已确认完毕！」

### 4. 框选截图 + 配图

案例块常见 `[📷 待补]`，8085 支持人工补图：

1. 左侧源文件拖拽框选
2. 「💾 保存截图」→ `user_crops/{stem}_p{页}_{时间戳}.png`
3. 截图列表点「复制」→ 粘贴 `[📷 user_crops/xxx.png]` 到正文
4. 「保存并重新嵌入」写库

与 `add_case_screenshots.py`（自动蓝框检测）互补：**8085 = 人工精确框选**。

### 5. 删除块 + 回收站

「删除此块」前：

- 导出到 `deleted_chunks/*.json`（含 **原始向量**）
- 从 ChromaDB 删除（检索立即失效）

回收站支持：

- 预览被删内容
- **♻ 恢复**：优先用原始向量写回（无需重算 embedding）
- **✗ 永久删除**：删 JSON，不可恢复

## 五、与飞书问答的衔接

| 8085 操作 | 飞书侧效果 |
|-----------|------------|
| 改正文并保存 | 向量/BM25 检索命中新文字 |
| 插入 `[📷 user_crops/xxx.png]` | 解析标签，**随答案发图** |
| 预览栏表格警告 | 提示改用纯文字或截图，避免飞书乱码 |
| 确认完成 | 仅管理标记，不影响检索逻辑 |

飞书解析规则：`[📷 路径]`（`feishu_ws_server_v11.py` 内正则匹配）。

## 六、适用数据范围

面向 **chroma_db 中所有已入库 source**，典型包括：

- `EHS案例/`（Vision LLM 路线）
- `设计部/案例/`、`客关部/排雷库` 等「案例」子目录
- 部分 `index_reviewed_md.py` 入库的常规文档（可二次修改）

**不适用**：尚未入库的 MD（请用 8084）。

## 七、与系统其他程序的关系

```
sync_kb_v11.py / index_reviewed_md.py
        ↓
    chroma_db
        ↓
  chunk_reviewer (8085)
        ├→ 改正文 + 重嵌入 → chroma_db
        ├→ 框选截图 → user_crops/
        ├→ 删块备份 → deleted_chunks/
        └→ 触发 → bm25_index.pkl → 飞书热加载
```

| 程序 | 与 8085 关系 |
|------|-------------|
| `add_case_screenshots.py` | 无人值守批量补图；8085 人工补图 |
| `reindex_design_cases.py` | 整文件批量重制；8085 单块精修 |
| `snapshot_kb.py` | 大批量改库前快照；8085 删块靠回收站 |
| `md_reviewer.py` (8084) | 入库前审 MD；8085 入库后审 chunk |
